# Governance Model

This document outlines the governance model for the **dumbmerch** project. Ideally, this document clarifies how decisions are made and how the community works together.

## 1. Overview

**dumbmerch** is an open-source project that is community-driven but led by a core team of maintainers. The goal is to foster a collaborative environment while ensuring the project remains focused and high-quality.

## 2. Roles and Responsibilities

### 2.1. Project Lead (Benevolent Dictator)
The Project Lead has the final say in all decisions ensuring the project's coherence and consistency.
- **Current Lead**: Dava

### 2.2. Maintainers
Maintainers are active contributors who have been given write access to the repository. They are responsible for:
- Reviewing Pull Requests.
- Triaging Issues.
- Merging approved changes.
- Maintaining the project roadmap.

### 2.3. Contributors
Anyone who submits a Pull Request or opens an issue. Contributors play a vital role in the project's growth.

## 3. Decision Making Process

1. **Consensus Seeking**: We aim for consensus among maintainers for technical decisions.
2. **Voting**: If consensus cannot be reached, a vote among maintainers may determine the outcome.
3. **Tie Breaking**: In the event of a deadlock, the Project Lead has the final casting vote.

## 4. Code Ownership

The codebase is owned by the organization/user acting as the repository owner. All contributions are licensed under the project's license (MIT).

## 5. Conflict Resolution

We expect all contributors to adhere to the [Code of Conduct](./CODE_OF_CONDUCT.md). Conflicts should be resolved through professional discussion. If a resolution cannot be reached, the Project Lead will mediate.
